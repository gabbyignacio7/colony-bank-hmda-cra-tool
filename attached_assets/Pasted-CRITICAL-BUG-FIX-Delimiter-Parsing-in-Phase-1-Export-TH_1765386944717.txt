CRITICAL BUG FIX: Delimiter Parsing in Phase 1 Export

THE PROBLEM:
The Excel export is dumping raw delimited strings into a single cell instead of splitting them into separate columns.

CURRENT OUTPUT (BROKEN):
- Cell A1 contains: "114030351~01-07-2025~02~183~1~03-05-2025~~~~~1~~1~Marcus Ted~Bozeman~..."
- All data crammed into Column A as delimited text
- Other columns are empty or have wrong headers

EXPECTED OUTPUT:
- Cell A1: "114030351"
- Cell B1: "01-07-2025"
- Cell C1: "02"
- Cell D1: "183"
- etc. - EACH VALUE IN ITS OWN COLUMN

ROOT CAUSE:
The code is reading delimited text files but NOT splitting on the delimiter before writing to Excel. It's treating the entire delimited line as a single cell value.

THE FIX:

1. DETECT AND SPLIT ALL DELIMITERS:
The input files use DIFFERENT delimiters:
- LaserPro .txt: Uses TILDE (~) delimiter
- Some files: Use PIPE (|) delimiter  
- Some files: Use TAB delimiter
- Some files: Use COMMA delimiter

Add this logic to detect and split:
```javascript
function parseDelimitedLine(line) {
  // Detect delimiter by checking which one appears most
  const delimiters = ['~', '|', '\t', ','];
  let bestDelimiter = ',';
  let maxCount = 0;
  
  for (const delim of delimiters) {
    const count = (line.match(new RegExp(delim === '|' ? '\\|' : delim, 'g')) || []).length;
    if (count > maxCount) {
      maxCount = count;
      bestDelimiter = delim;
    }
  }
  
  return line.split(bestDelimiter);
}
```

2. APPLY SPLITTING WHEN READING FILES:
When reading LaserPro .txt files:
- Read each line
- Split by detected delimiter (likely ~)
- Map each split value to the corresponding column
- Skip row 1 (metadata)

3. APPLY SPLITTING WHEN WRITING EXCEL:
The xlsx export must create SEPARATE CELLS, not dump a string:
```javascript
// WRONG - dumps entire string into one cell
worksheet['A1'] = { v: "value1~value2~value3" };

// CORRECT - each value in separate cell
worksheet['A1'] = { v: "value1" };
worksheet['B1'] = { v: "value2" };
worksheet['C1'] = { v: "value3" };
```

4. FILE PARSING RULES:
LaserPro (.txt):
- Delimiter: TILDE (~)
- Skip row 1 (metadata: "1|Colony Bank|2025|4|Jonathan Hester|...")
- Data starts row 2
- Split each line by ~ into array of values
